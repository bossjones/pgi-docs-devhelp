A Gdk.Cursor represents a cursor. Its contents are private.

Creates a new cursor from the set of builtin cursors for the default display. See Gdk.Cursor.new_for_display ().

To make the cursor invisible, use Gdk.CursorType.BLANK_CURSOR.

Creates a new cursor from the set of builtin cursors. Some useful ones are: - ![](right_ptr.png) Gdk.CursorType.RIGHT_PTR (right-facing arrow) - ![](crosshair.png) Gdk.CursorType.CROSSHAIR (crosshair) - ![](xterm.png) Gdk.CursorType.XTERM (I-beam) - ![](watch.png) Gdk.CursorType.WATCH (busy) - ![](fleur.png) Gdk.CursorType.FLEUR (for moving objects) - ![](hand1.png) Gdk.CursorType.HAND1 (a right-pointing hand) - ![](hand2.png) Gdk.CursorType.HAND2 (a left-pointing hand) - ![](left_side.png) Gdk.CursorType.LEFT_SIDE (resize left side) - ![](right_side.png) Gdk.CursorType.RIGHT_SIDE (resize right side) - ![](top_left_corner.png) Gdk.CursorType.TOP_LEFT_CORNER (resize northwest corner) - ![](top_right_corner.png) Gdk.CursorType.TOP_RIGHT_CORNER (resize northeast corner) - ![](bottom_left_corner.png) Gdk.CursorType.BOTTOM_LEFT_CORNER (resize southwest corner) - ![](bottom_right_corner.png) Gdk.CursorType.BOTTOM_RIGHT_CORNER (resize southeast corner) - ![](top_side.png) Gdk.CursorType.TOP_SIDE (resize top side) - ![](bottom_side.png) Gdk.CursorType.BOTTOM_SIDE (resize bottom side) - ![](sb_h_double_arrow.png) Gdk.CursorType.SB_H_DOUBLE_ARROW (move vertical splitter) - ![](sb_v_double_arrow.png) Gdk.CursorType.SB_V_DOUBLE_ARROW (move horizontal splitter) - Gdk.CursorType.BLANK_CURSOR (Blank cursor).

Gdk.Cursor

Creates a new cursor from a pixbuf.

Not all GDK backends support RGBA cursors. If they are not supported, a monochrome approximation will be displayed. The functions Gdk.Display.supports_cursor_alpha () and Gdk.Display.supports_cursor_color () can be used to determine whether RGBA cursors are supported; Gdk.Display.get_default_cursor_size () and Gdk.Display.get_maximal_cursor_size () give information about cursor sizes.

If x or y are `-1`, the pixbuf must have options named “x_hot” and “y_hot”, resp., containing integer values between `0` and the width resp. height of the pixbuf. (Since: 3.0)

On the X backend, support for RGBA cursors requires a sufficently new version of the X Render extension.

Returns:a GdkPixbuf.Pixbuf representing cursor, or None
Return type:GdkPixbuf.Pixbuf or None

Returns a GdkPixbuf.Pixbuf with the image used to display the cursor.

Note that depending on the capabilities of the windowing system and on the cursor, GDK may not be able to obtain the image data. In this case, None is returned.
