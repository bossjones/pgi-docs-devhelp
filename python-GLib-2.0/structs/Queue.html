class GLib.Queue

Contains the public fields of a [Queue][glib-Double-ended-Queues].

clear()

Removes all the elements in queue. If queue elements contain dynamically-allocated memory, they should be freed first.

New in version 2.14.

free()

Frees the memory allocated for the GLib.Queue. Only call this function if queue was created with GLib.Queue.new (). If queue elements contain dynamically-allocated memory, they should be freed first.

If queue elements contain dynamically-allocated memory, you should either use GLib.Queue.free_full () or free them manually first.

free_full(free_func)
Parameters:free_func (GLib.DestroyNotify) – the function to be called to free each element’s data

Convenience method, which frees all the memory used by a GLib.Queue, and calls the specified destroy function on every element’s data.

New in version 2.32.

New in version 2.4.

index(data)
Parameters:data (object) – the data to find
Returns:the position of the first element in queue which contains data, or -1 if no element in queue contains data
Return type:int

Returns the position of the first element in queue which contains data.

New in version 2.4.

init()

A statically-allocated GLib.Queue must be initialized with this function before it can be used. Alternatively you can initialize it with #G_QUEUE_INIT. It is not necessary to initialize queues created with GLib.Queue.new ().

New in version 2.14.

is_empty()
Returns:True if the queue is empty
Return type:bool

Returns True if the queue is empty.

Parameters:data (object) – the data for the new element.

push_nth(data, n)
Parameters:
  • data (object) – the data for the new element
  • n (int) – the position to insert the new element. If n is negative or larger than the number of elements in the queue, the element is added to the end of the queue.

Inserts a new element into queue at the given position.

New in version 2.4.
