static get_object_by_guid(mgd, guid)
Parameters:
  • mgd (Midgard.Connection) – Midgard.Connection instance
  • guid (str) – guid which identifies object to look for
Returns:

Midgard.Object derived new instance or None on failure
Return type:

Midgard.Object

Creates new instance of the class defined in Midgard Schema.

Cases to return None:

Given guid is not a valid guid (MGD_ERR_NOT_EXISTS)

There’s no object identified by given guid (MGD_ERR_NOT_EXISTS)

Object identified by given guid is deleted (MGD_ERR_OBJECT_DELETED)

Object identified by given guid is purged (MGD_ERR_OBJECT_PURGED)

Object identified by given guid doesn’t provide ‘metadata’ or ‘deleted’ property (MGD_ERR_INVALID_PROPERTY)

New in version 10.05.

static get_object_by_path(mgd, classname, path)
Parameters:
  • mgd (Midgard.Connection) – Midgard.Connection instance
  • classname (str) – name of the class, new instance should be created for
  • path (str) – path which identifies object
Returns:

Midgard.Object derived, new classname instance or None
Return type:

Midgard.Object

Get object by path. Path elements are objects’ names. To get top object with empty name use “/” path. Also empty name is allowed in given path: “/A/B//D”.

Cases to return None:

Object identified by given path doesn’t exist (MGD_ERR_NOT_EXISTS)

Given classname doesn’t support tree functionality (MGD_ERR_NOT_INTERNAL)

Given classname doesn’t provide ‘id’ or unique named property (MGD_ERR_NOT_INTERNAL)

New in version 10.05.
