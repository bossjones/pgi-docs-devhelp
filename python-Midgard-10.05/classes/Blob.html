Invokes Midgard.Blob.new () and it’s designed for language bindings, in whic, that function can not be invoked explicitly.

New in version 10.05.1.

static new(attachment, encoding)
Parameters:
  • attachment (Midgard.Object) – Midgard.Object of MIDGARD_TYPE_ATTACHMENT type.
  • encoding (str or None) – file encoding
Returns:

newly instatiated Midgard.Blob object or None on failure
Return type:

Midgard.Blob

Default encoding is UTF-8. Set None encoding if such is required.

Instatiate new Midgard Blob object for the given midgard_attachment object.

This constructor defines new relative path for attachment, if midgard_attachment is associated with midgard_blob and its location is empty. In any other case, location is not changed.

Check if file associated with midgard_blob exists. This function will also return False, if file is not yet associated.

get_handler(mode)
Parameters:mode (str) – fopen mode (r, w, a, b). Default is ‘w’.
Raises:GLib.GError
Returns:GLib.IOChannel or None
Return type:GLib.IOChannel

The main idea is to get file handler. On C level it returns GLib.IOChannel, but language bindings could return typical file handler or anything else which is needed for particular language.

Returned channel is owned by midgard_blob and should not be freed or unreferenced.

get_path()
Returns:absolute path or None if path is not set yet.
Return type:str

Returned path is owned by midgard_blob and should not be freed. It basically contains blobdir and relative file’s location.

read_content()
Returns:content of the file, or None on failure
bytes_read:number of bytes read
Return type:(str, bytes_read: int)

Returned content should be freed when no longer needed. bytes_read holds size of returned content.

This function should be used to get content of small files. For large and huge ones Midgard.Blob.get_handler should be used to get file handle.

remove_file()
Raises:GLib.GError
Returns:True on success, False otherwise
Return type:bool

Deletes a file which is associated with blob and located at attachment’s location which is initialized for blob. Midgard.Blob.exists should be invoked if file may be already deleted, for example when one file is shared among many attachments.
