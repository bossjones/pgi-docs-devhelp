class GstFft.FFTF64

GstFft.FFTF64 provides a FFT implementation and related functions for 64 bit float samples. To use this call GstFft.FFTF64.new () for allocating a GstFft.FFTF64 instance with the appropriate parameters and then call GstFft.FFTF64.fft () or GstFft.FFTF64.inverse_fft () to perform the FFT or inverse FFT on a buffer of samples.

After use free the GstFft.FFTF64 instance with GstFft.FFTF64.free ().

For the best performance use GstFft.fft_next_fast_length () to get a number that is entirely a product of 2, 3 and 5 and use this as the len parameter for GstFft.FFTF64.new ().

The len parameter specifies the number of samples in the time domain that will be processed or generated. The number of samples in the frequency domain is len /2 + 1. To get n samples in the frequency domain use 2*n - 2 as len.

Before performing the FFT on time domain data it usually makes sense to apply a window function to it. For this GstFft.FFTF64.window () can comfortably be used.

Be aware, that you can’t simply run GstFft.FFTF32.inverse_fft () on the resulting frequency data of GstFft.FFTF32.fft () to get the original data back. The relation between them is iFFT (FFT (x)) = x * nfft where nfft is the length of the FFT. This also has to be taken into account when calculation the magnitude of the frequency data.

fft(timedata, freqdata)
Parameters:
  • timedata (float) – Buffer of the samples in the time domain
  • freqdata (GstFft.FFTF64Complex) – Target buffer for the samples in the frequency domain

This performs the FFT on timedata and puts the result in freqdata.

timedata must have as many samples as specified with the len parameter while allocating the GstFft.FFTF64 instance with GstFft.FFTF64.new ().

freqdata must be large enough to hold len /2 + 1 GstFft.FFTF64Complex frequency domain samples.

free()

This frees the memory allocated for self.

inverse_fft(freqdata, timedata)
Parameters:
  • freqdata (GstFft.FFTF64Complex) – Buffer of the samples in the frequency domain
  • timedata (float) – Target buffer for the samples in the time domain

This performs the inverse FFT on freqdata and puts the result in timedata.

freqdata must have len /2 + 1 samples, where len is the parameter specified while allocating the GstFft.FFTF64 instance with GstFft.FFTF64.new ().

timedata must be large enough to hold len time domain samples.

window(timedata, window)
Parameters:
  • timedata (float) – Time domain samples
  • window (GstFft.FFTWindow) – Window function to apply

This calls the window function window on the timedata sample buffer.
