add_priority(map, from_, to)

  • from_ (int) – the priority to map to to
  • to (int) – the priority to map from to
Returns:

True if the new priority mapping was successfully added to the list, False if error
Return type:

bool

Adds a priority mapping to the NetworkManager.SettingVlan :ingress-priority-map or NetworkManager.SettingVlan :egress-priority-map properties of the setting. If from is already in the given priority map, this function will overwrite the existing entry with the new to.

If map is NetworkManager.VlanPriorityMap.INGRESS_MAP then from is the incoming 802.1q VLAN Priority Code Point (PCP) value, and to is the Linux SKB priority value.

If map is NetworkManager.VlanPriorityMap.EGRESS_MAP then from is the Linux SKB priority value and to is the outgoing 802.1q VLAN Priority Code Point (PCP) value.

Property Details

NetworkManager.SettingVlan.props.flags
Name:flags
Type:int
Default Value:0
Flags:r/w/c

One or more flags which control the behavior and features of the VLAN interface. Flags include NetworkManager.VlanFlags.REORDER_HEADERS (reordering of output packet headers), NetworkManager.VlanFlags.GVRP (use of the GVRP protocol), and NetworkManager.VlanFlags.LOOSE_BINDING (loose binding of the interface to its master device’s operating state).

NetworkManager.SettingVlan.props.id
Name:id
Type:int
Default Value:0
Flags:r/w/c

The VLAN identifier that the interface created by this connection should be assigned.

NetworkManager.SettingVlan.props.interface_name
Name:interface-name
Type:str
Default Value:None
Flags:r/w/c

If given, specifies the kernel name of the VLAN interface. If not given, a default name will be constructed from the interface described by the parent interface and the NetworkManager.SettingVlan :id property, eg “eth2.1”. The parent interface may be given by the NetworkManager.SettingVlan :parent property or by the NetworkManager.SettingWired :mac-address property of an NetworkManager.SettingWired setting.

NetworkManager.SettingVlan.props.parent
Name:parent
Type:str
Default Value:None
Flags:r/w/c

If given, specifies the parent interface name or parent connection UUID from which this VLAN interface should be created. If this property is not specified, the connection must contain an NetworkManager.SettingWired setting with a NetworkManager.SettingWired :mac-address property.
