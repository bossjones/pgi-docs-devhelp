Properties

Inherited:NetworkManager.Setting (1)
Name Type Flags Short Description
baud int r/w/c If non-zero, instruct pppd to set the serial port to the specified baudrate. This value should normally be left as 0 to automatically choose the speed.
crtscts bool r/w/c If True, specify that pppd should set the serial port to use hardware flow control with RTS and CTS signals. This value should normally be set to False.
lcp-echo-failure int r/w/c If non-zero, instruct pppd to presume the connection to the peer has failed if the specified number of LCP echo-requests go unanswered by the peer. The ‘lcp-echo-interval’ property must also be set to a non-zero value if this property is used.
lcp-echo-interval int r/w/c If non-zero, instruct pppd to send an LCP echo-request frame to the peer every n seconds (where n is the specified value). Note that some PPP peers will respond to echo requests and some will not, and it is not possible to autodetect this.
mppe-stateful bool r/w/c If True, stateful MPPE is used. See pppd documentation for more information on stateful MPPE.
mru int r/w/c If non-zero, instruct pppd to request that the peer send packets no larger than the specified size. If non-zero, the MRU should be between 128 and 16384.
mtu int r/w/c If non-zero, instruct pppd to send packets no larger than the specified size.
no-vj-comp bool r/w/c If True, Van Jacobsen TCP header compression will not be requested.
noauth bool r/w/c If True, do not require the other side (usually the PPP server) to authenticate itself to the client. If False, require authentication from the remote side. In almost all cases, this should be True.
nobsdcomp bool r/w/c If True, BSD compression will not be requested.
nodeflate bool r/w/c If True, ‘deflate’ compression will not be requested.
refuse-chap bool r/w/c If True, the CHAP authentication method will not be used.
refuse-eap bool r/w/c If True, the EAP authentication method will not be used.
refuse-mschap bool r/w/c If True, the MSCHAP authentication method will not be used.
refuse-mschapv2 bool r/w/c If True, the MSCHAPv2 authentication method will not be used.
refuse-pap bool r/w/c If True, the PAP authentication method will not be used.
require-mppe bool r/w/c If True, MPPE (Microsoft Point-to-Point Encrpytion) will be required for the PPP session. If either 64-bit or 128-bit MPPE is not available the session will fail. Note that MPPE is not used on mobile broadband connections.
require-mppe-128 bool r/w/c If True, 128-bit MPPE (Microsoft Point-to-Point Encrpytion) will be required for the PPP session, and the ‘require-mppe’ property must also be set to True. If 128-bit MPPE is not available the session will fail.
