Class Details

class WebKit.SpellChecker

Bases: GObject.GInterface

WebKit.SpellChecker provides APIs for the spell checking functionality used internally by WebKit to perform spell checking in editable areas. This can be used, for example, by browsers to implement custom spell checking context menus or sophisticated auto-correct features.

check_spelling_of_string(string)
Parameters:string (str) – the string to check for misspellings
Returns:
misspelling_location:
 a pointer to an integer to store the location of the first misspelling
misspelling_length:
 a pointer to an integer to store the length of the first misspelling
Return type:(misspelling_location: int, misspelling_length: int)

Checks string for misspellings using checker, storing the location and length of the first misspelling in misspelling_location and misspelling_length respectively.

New in version 1.5.1.

get_autocorrect_suggestions_for_misspelled_word(word)
Parameters:word (str) – a misspelled word
Returns:the suggestion for the autocorrection of word
Return type:str

Returns a suggestion for a word to use in an “autocorrect” feature.

New in version 1.5.1.

get_guesses_for_word(word, context)
Parameters:
  • word (str) – the misspelled word
  • context (str or None) – the surrounding context of the misspelled word
Returns:

a newly allocated None-terminated array of suggested corrections for a misspelled word word. Free it with GLib.strfreev when done with it.
Return type:

[str]

Returns a None-terminated array of guesses for corrections of the misspelled word word.

New in version 1.5.1.

ignore_word(word)
Parameters:word (str) – the word to ignore

Instructs the checker to ignore word as a misspelling for this session.

New in version 1.5.1.

learn_word(word)
Parameters:word (str) – the word to learn

Instructs the checker to add word to its dictionary as a properly spelled word. The word will be learned permanently in the user’s personal dictionary.

New in version 1.5.1.

update_spell_checking_languages(languages)
Parameters:languages (str or None) – a string of languages to use for checker

Sets languages as the list of languages to use by checker. The accepted format is a list of comma (‘,’) separated language codes of the form ‘en_US’, ie, language_VARIANT.

New in version 1.5.1.
