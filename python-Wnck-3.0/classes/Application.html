Class Details

class Wnck.Application(**kwargs)

Bases: GObject.Object

The Wnck.Application struct contains only private fields and should not be directly accessed.

static get(xwindow)
Parameters:xwindow (int) – the X window ID of a group leader.
Returns:the Wnck.Application corresponding to xwindow, or None if there no such Wnck.Application could be found. The returned Wnck.Application is owned by libwnck and must not be referenced or unreferenced.
Return type:Wnck.Application

Gets the Wnck.Application corresponding to the group leader with xwindow as X window ID.

get_icon()
Returns:the icon for app. The caller should reference the returned GdkPixbuf.Pixbuf if it needs to keep the icon around.
Return type:GdkPixbuf.Pixbuf

Gets the icon to be used for app. If no icon is set for app, a suboptimal heuristic is used to find an appropriate icon. If no icon was found, a fallback icon is used.

get_icon_is_fallback()
Returns:True if the icon for app is a fallback, False otherwise.
Return type:bool

Gets whether a default fallback icon is used for app (because none was set on app ).

get_icon_name()
Returns:the icon name of app, or a fallback icon name if no icon name is available.
Return type:str

Gets the icon name of app (to be used when app is minimized). Since there is no way to properly find this name, various suboptimal heuristics are used to find it.

get_mini_icon()
Returns:the mini-icon for app. The caller should reference the returned GdkPixbuf.Pixbuf if it needs to keep the mini-icon around.
Return type:GdkPixbuf.Pixbuf

Gets the mini-icon to be used for app. If no mini-icon is set for app, a suboptimal heuristic is used to find an appropriate icon. If no mini-icon was found, a fallback mini-icon is used.

get_n_windows()
Returns:the number of Wnck.Window belonging to app.
Return type:int

Gets the number of Wnck.Window belonging to app.

get_name()
Returns:the name of app, or a fallback name if no name is available.
Return type:str

Gets the name of app. Since there is no way to properly find this name, various suboptimal heuristics are used to find it. GTK+ should probably have a function to allow applications to set the _NET_WM_NAME property on the group leader as the application name, and the EWMH should say that this is where the application name goes.

get_pid()
Returns:the process ID of app, or 0 if none is available.
Return type:int

Gets the process ID of app.

get_startup_id()
Returns:the startup sequence ID used for startup notification of app, or None if none is available.
Return type:str

Gets the startup sequence ID used for startup notification of app.

New in version 2.2.

get_windows()
Returns:the list of Wnck.Window belonging to app, or None if the application contains no window. The list should not be modified nor freed, as it is owned by app.
Return type:[Wnck.Window]

Gets the list of Wnck.Window belonging to app.

get_xid()
Returns:the X window ID of the group leader window for app.
Return type:int

Gets the X window ID of the group leader window for app.
