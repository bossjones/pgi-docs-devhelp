class GWeather.Timezone

A timezone.

Timezones are global to the #GWeatherWorld; they can be gotten by passing GWeather.Timezone.get_by_tzid () with a tzid like “America/New_York” or “Europe/London”.

static get_by_tzid(tzid)
Parameters:tzid (str) – A timezone identifier, like “America/New_York” or “Europe/London”
Returns:A GWeather.Timezone. This object belongs to GWeather, do not unref it.
Return type:GWeather.Timezone

Get the GWeather.Timezone for tzid.

New in version 3.12.

get_dst_offset()
Returns:zone ‘s daylight/summer time offset, in minutes
Return type:int

Gets zone ‘s daylight/summer time offset from UTC, in minutes. Eg, a value of 120 would indicate “GMT+2”. This is only meaningful if GWeather.Timezone.has_dst () returns True.

get_name()
Returns:zone ‘s name
Return type:str

Gets zone ‘s name; a translated, user-presentable string.

Note that the returned name might not be unique among timezones, and may not make sense to the user unless it is presented along with the timezone’s country’s name (or in some context where the country is obvious).

get_offset()

Return type:int

Gets zone ‘s standard offset from UTC, in minutes. Eg, a value of 120 would indicate “GMT+2”.

has_dst()
