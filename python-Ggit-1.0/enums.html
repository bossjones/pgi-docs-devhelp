class Ggit.ConfigLevel

Bases: GObject.GEnum

Priority level of a config file. These priority levels correspond to the natural escalation logic (from higher to lower) when searching for config entries in git.git.

SYSTEM = 1

System-wide configuration file.

XDG = 2

XDG compatible configuration file (.config/git/config).

GLOBAL = 3
LOCAL = 4

Repository specific configuration file.

class Ggit.DiffLineType

Bases: GObject.GEnum

These values describe where a line came from and will be passed to the Ggit.DiffLineCallback when iterating over a diff.

The Ggit.DiffLineType.FILE_HDR, Ggit.DiffLineType.HUNK_HDR and Ggit.DiffLineType.BINARY values are only sent when the diff is being printed.

CONTEXT = 32

line is part of the context.

ADDITION = 43

line that was added.

DELETION = 45

line that was removed.

DEL_EOFNL = 60

LF was removed at end of file.

CONTEXT_EOFNL = 61

Both files have no LF at end.

ADD_EOFNL = 62

LF was added at end of file.

BINARY = 66

is binary.

FILE_HDR = 70

the file header.

HUNK_HDR = 72

the hunk header.

class Ggit.ResetType

Bases: GObject.GEnum

Describes the type of reset to perform.

SOFT = 1

The head will be moved to the commit.

MIXED = 2

the head will be moved to the commit and the index will be replaced with the content of the commit tree.

HARD = 3

MIXED plus changes in working tree discarded.

class Ggit.StatusShow

Bases: GObject.GEnum

Show options for Ggit.Repository.file_status_foreach. Determines which files are included in the status.

INDEX_AND_WORKDIR = 0

receive one callback for each file, even if the file is in both the index and the workdir (combining Ggit.StatusFlags ).

INDEX_ONLY = 1

only receive callbacks for files in the index.

WORKDIR_ONLY = 2

only receive callbacks for files in the workdir.

class Ggit.SubmoduleUpdate

Bases: GObject.GEnum

Describes how a submodule should be updated.

RESET = -1

reset to on-disk value.

CHECKOUT = 1

checkout the submodule.

REBASE = 2

rebase the current branch onto the commit recorded in the superproject.

MERGE = 3

merge the commit recorded in the superproject into the current branch.

NONE = 4

do not update this submodule.
